Introduction

Object-oriented programming (OOP) is a fundamental programming paradigm used by nearly every developer at some point in their career.

It is a computer programming model that organizes software design around data, or objects, rather than functions and logic.

Class and Object

Classes and objects are the two main aspects of object-oriented programming.

A class is a template for objects, and an object is an instance of a class. Class have attributes and functions. Somtimes we called it as properties and methods respectively.

For example, car is a class. Toyota Vios is an object. Perodua Myvi can be an object too. Color is the attributes of the class. Speed Up is the functions of the car. Below code is the example: 

                    
                    class Car
                    {
                        string color; // attributes
                        int maxSpeed; // attributes
                    
                        public void fullThrottle() // functions
                        {
                            Console.WriteLine("The car is going as fast as it can!");
                        }
                    
                        static void Main(string[] args)
                        {
                            Car myMyvi = new Car();
                            myMyvi.color = "red";
                            myMyvi.maxSpeed = 200;
                            myMyvi.fullThrottle(); // Call the method
                        }
                    }
Abstraction

Abstraction is a fundamental concept in object-oriented programming that helps to hide the complex implementation details and show only the necessary features of an object.

In C#, you can achieve abstraction using abstract classes and interfaces. 

                    
                    // Abstract class
                    public abstract class Shape
                    {
                        public abstract double CalculateArea();
                    }
                                        
                    // Derived class
                    public class Rectangle : Shape
                    {
                        public double Width { get; set; }
                        public double Height { get; set; }
                        public override double CalculateArea() => Width * Height;
                    }
                    
                    class Program
                    {
                        static void Main()
                        {
                            Shape rectangle = new Rectangle { Width = 5, Height = 10 };
                            Console.WriteLine($"Area of rectangle: {rectangle.CalculateArea()}");
                        }
                    }
Encapsulation

Encapsulation is another core principle of object-oriented programming. It involves bundling the data (fields) and methods (functions) that operate on the data into a single unit (class) and restricting access to some of the object's components.

This helps to protect the integrity of the data and encapsulate the internal workings of the class. 

                    
                    public class Person
                    {
                        private string name;
                        private int age;
                    
                        public string Name { get; set; }
                        public int Age
                        {
                            get => age;
                            set => age = (value >= 0) ? value : throw new ArgumentException("Age cannot be negative.");
                        }
                    
                        public void Display() => Console.WriteLine($"Name: {Name}, Age: {Age}");
                    }
                    
                    class Program
                    {
                        static void Main()
                        {
                            var person = new Person { Name = "John Doe", Age = 30 };
                            person.Display();
                        }
                    }
Inheritance

Inheritance allows a class to inherit members (fields, methods, properties) from another class.

This helps to create a hierarchical relationship between classes and promotes code reuse. 

                    
                    // Base class
                    public class Animal
                    {
                        public void Eat() => Console.WriteLine("Animal is eating.");
                    }
                                        
                    // Derived class
                    public class Dog : Animal
                    {
                        public void Bark() => Console.WriteLine("Dog is barking.");
                    }
                    
                    class Program
                    {
                        static void Main()
                        {
                            // Create an instance of Dog
                            Dog dog = new Dog();
                                        
                            // Call inherited method
                            dog.Eat();
                                        
                            // Call method specific to Dog
                            dog.Bark();
                        }
                    }
Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common base class.

It is a fundamental concept in object-oriented programming that enables methods to be used in different ways depending on the object calling them.

There are two main types of polymorphism in C#:

  • Compile-time polymorphism (method overloading)
  • Runtime polymorphism (method overriding)
                    
                    public class Printer
                    {
                        // Method to print integer
                        public void Print(int number) => Console.WriteLine($"Printing integer: {number}");
                    
                        // Method to print string
                        public void Print(string text) => Console.WriteLine($"Printing string: {text}");
                    }
                    
                    class Program
                    {
                        static void Main()
                        {
                            var printer = new Printer();
                    
                            // Call overloaded methods
                            printer.Print(123);
                            printer.Print("Hello, World!");
                        }
                    }
                    
                
                    
                    // Base class
                    public class Animal
                    {
                        public virtual void Speak() => Console.WriteLine("Animal makes a sound.");
                    }
                    
                    // Derived class
                    public class Dog : Animal
                    {
                        public override void Speak() => Console.WriteLine("Dog barks.");
                    }
                                        
                    class Program
                    {
                        static void Main()
                        {
                            // Create objects of derived classes
                            Animal myDog = new Dog();
                                        
                            // Call the overridden methods
                            myDog.Speak(); // Outputs: Dog barks.
                        }
                    }
Why do we need OOP

  • To make the development and maintenance of projects more effortless.
  • To provide the feature of data hiding that is good for security concerns.
  • We can solve real-world problems if we are using object-oriented programming.
  • It ensures code reusability.
  • It lets us write generic code: which will work with a range of data, so we don't have to write basic stuff over and over again.
